# Input Completeness Check

Read `../../core/input-standard.md` and the relevant sport folder in `../../sports/`.

## Workflow
1. Match the setup to the correct sport and market.
2. List required inputs for that context.
3. Mark each required input as present, missing, stale, or unclear.
4. Distinguish recoverable gaps from blocking gaps.

## Return
- confirmed inputs
- missing inputs
- blocking gaps
- provisional completeness status
