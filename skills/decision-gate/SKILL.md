# Decision Gate

Read `../../core/decision-gate.md`.

## Workflow
1. Review `data quality`, `edge quality`, `risk load`, and red flags together.
2. Apply the project gate rules.
3. Prefer discipline over action bias.
4. If the verdict is `watchlist`, define the trigger that could change it.

## Return
- final verdict
- short reason
- trigger conditions when relevant
