# sports/f1/overview.md

## Objective
This module covers F1 analysis using structural timing modes.

## Core markets
- qualifying H2H
- race H2H
- top 6
- top 10

## Selective markets
- winner
- podium

## Required inputs
- event
- timing mode
- market
- side
- price and timestamp
- weather status
- penalties or news
- inputs compatible with the active mode

## Central principle
F1 does not use a single static workflow. The system must first identify the correct timing mode.
