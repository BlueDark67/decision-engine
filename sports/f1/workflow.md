# sports/f1/workflow.md

## Main workflow
1. Define event, market, side, and price.
2. Identify the timing mode.
3. Audit whether the required inputs for that mode exist.
4. Build the performance and strategy read.
5. Translate the read into the market.
6. Assess `data quality`, `edge quality`, and `risk load`.
7. Audit red flags.
8. Pass through the decision gate.

## Baseline factors
- one-lap pace
- race pace
- degradation
- track position
- overtaking difficulty
- penalties
- weather
- reliability

## Red flags
- unresolved penalties
- unstable weather
- upgrade uncertainty
- sprint weekend with thin data
- thesis relying too heavily on reputation rather than pace evidence
