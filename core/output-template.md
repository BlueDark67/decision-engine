# core/output-template.md

## Objective
This document details how to fill the final schema.

## Fields
### `Analysis Mode`
Identify the mode used:

- football `strong-stat-coverage` or `weak-stat-coverage`
- NBA core workflow
- applicable F1 timing mode

### `Key Supporting Factors`
List only the factors that genuinely support the read.
Avoid noise.

### `Red Flags`
List only the flags that matter to the decision.

### `Why This Verdict`
Explain the final decision in short, disciplined language.

### `What Could Change The Verdict`
Indicate objective triggers:

- lineups
- injury confirmation
- price movement
- qualifying
- weather
- penalties

### `Stability / Confidence Note`
Close with a short note about the stability of the verdict.

Examples:

- `Stable: the core inputs are confirmed and the context is relatively clean.`
- `Moderately stable: the thesis is valid, but it still depends on one key confirmation.`
- `Fragile: the read exists, but it is sensitive to still-unstable inputs.`
